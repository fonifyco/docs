## Hoş Geldiniz

Fonify.co API'si ile AI destekli sesli asistan sisteminizi programatik olarak yönetebilirsiniz. API, RESTful standartlarını takip eder ve JSON formatında veri alışverişi yapar.

API dokümantasyonu iki şekilde oluşturulabilir: [OpenAPI](https://mintlify.com/docs/api-playground/openapi/setup) ve [MDX components](https://mintlify.com/docs/api-playground/mdx/configuration). Bu dokümantasyon için aşağıdaki OpenAPI spesifikasyonunu kullanıyoruz.

## Temel Bilgiler

### Base URL

Tüm API istekleri aşağıdaki base URL üzerinden yapılır:

```
https://api.fonify.co/v1
```

### İçerik Tipi

Tüm istekler ve yanıtlar JSON formatındadır:

```
Content-Type: application/json
```

## Kimlik Doğrulama (Authentication)

Tüm API endpoint'leri Bearer token kullanılarak kimlik doğrulaması gerektirir. API anahtarınızı almak için [Fonify.co platformu](https://fonify.co)'na giriş yapın.

### Bearer Token Kullanımı

Her istekte `Authorization` header'ında Bearer token'ınızı gönderin:

```bash
Authorization: Bearer YOUR_API_KEY
```

### Örnek İstek

```bash
curl -X GET https://api.fonify.co/v1/agents \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"
```

### OpenAPI Spesifikasyonu

API'miz OpenAPI spesifikasyonunda aşağıdaki şekilde tanımlanmıştır:

```json
"security": [
  {
    "bearerAuth": []
  }
]
```

## Rate Limiting

API kullanımı için aşağıdaki limitler geçerlidir:

- **Ücretsiz Plan**: Dakikada 60 istek
- **Pro Plan**: Dakikada 300 istek
- **Enterprise Plan**: Özel limitler

Rate limit aşıldığında `429 Too Many Requests` hatası alırsınız.

## Hata Kodları

API aşağıdaki HTTP durum kodlarını kullanır:

| Kod | Açıklama |
|-----|----------|
| 200 | Başarılı istek |
| 201 | Kaynak başarıyla oluşturuldu |
| 400 | Hatalı istek parametreleri |
| 401 | Kimlik doğrulama hatası |
| 403 | Erişim yetkisi yok |
| 404 | Kaynak bulunamadı |
| 429 | Rate limit aşıldı |
| 500 | Sunucu hatası |
